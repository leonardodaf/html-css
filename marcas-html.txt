FORMULÁRIOS : 

form - tag que cria a delimitação do que é o formulário
exemplo: <form> <p> digite seu nome: </p> </form>

action - fica dentro da tag form, é a ação de pra onde será enviado/armazenado o formulário (a mais popular é em .php)
exemplo: <form action= "cadastro.php"> </form>

autocomplete - serve para autocompletar ou não a caixa de texto, fica dentro do form
exemplo : <form action= "cadastro.php" autocomplete="off OU on"

autocomplete - serve para autocompletar ou não a caixa de texto, só que dentro do input lhe dá opções do navegador identificar o que exatamente é para completar
exemplo: <input type="text" name="nome" id="inome" autocomplete="username"> </input>
exemplo2: <input type="text" name="senha" id="isenha" autocomplete="current-password"> </input>

input - caixa de texto onde se adiciona as informações de vários tipos ou types como é chamado, do formulário
exemplo de input de texto: <form> <p> digite seu nome: </p> <input> type="text" name="nome" id="inome" </input> </form>
exemplo de input de botão de envio:<input> type="submit" value="enviar" </input> (o nome que fica dentro do botão, escrever dentro de VALUE)

name - é mais útil pra reconhecimento do html e php. serve para identificar
id - é mais útil para o LABEL, e para liguangens como javascript, por exemplo. serve para identificar

label - relaciona o texto com a caixa de texto que deseja envolver
exemplo: <form> <p> <label for="inome"> digite seu nome: </label> </p> <input> type="text" name="nome" id="inome" </input> </form>
OBS : no FOR, coloca o ID do item que deseja relacionar com a caixa de texto, deste modo, relacionando "digite seu nome" com o INPUT de nome

method - seleciona o metódo de envio de formulário, GET ou POST

get - método padrão de envio de formulário, aparece as informações no link e só pode ser preenchido por apróx. 3000 letras (recomendado pra buscas)

post - usar quando os dados a serem enviados forem mais sensíveis, como cartão, imagem ou qualquer outro arquivo, senha, endereço, cep... (mais recomendado pro geral)

OBS SOBRE POST : n traz proteção de fato, só n deixa 100% explícito em link como em GET, é recomendado estudar sobre https para proteger dados

required - serve para preenchimento obrigatório daquele campo específico, tem q colocar dentro da tag input

minlenght - mínimo de caracteres que pode no preenchimento de campo, tem q colocar dentro da tag input
maxlenghtn- máximo de caracteres que pode no preenchimento de campo, tem q colocar dentro da tag input

size - define quantos caracteres por vez ele irá mostrar na caixa de texto

placeholder - mensagem dentro da caixa de texto do que precisa ser digitado ali, tem q colocar dentro da tag input
